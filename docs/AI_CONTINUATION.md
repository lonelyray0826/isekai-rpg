# AI 接續開發文件

最後更新：2026-03-13  
目前基底版本：Beta v1.9 關閉 D20、回歸傳統 RPG 判定版

## 1. 專案定位
- 類型：中世紀異世界、離線網頁單機版、文字 RPG
- 執行方式：直接開啟 `index.html`
- 核心限制：**不可依賴伺服器**、**可用 `file://` 直接執行**、**數據類內容以 `.db` 管理**、**系統拆成多個 JS 檔維護**
- 當前玩法骨架：自由探索、戰鬥、任務、公會委託、部位解剖、傳統 RPG 百分比判定、生存項、副職業

## 2. 技術架構
### 2.1 前端
- 純前端：HTML + CSS + JavaScript
- 主入口：`index.html`
- 系統檔案位於：`assets/`

### 2.2 資料庫
- 主資料庫：`data/game_content.db`
- 前端離線快取：`assets/db-cache.js`
- 原因：瀏覽器直接用 `file://` 開啟時，不適合直接讀 SQLite，所以 `.db` 是**權威資料來源**，`db-cache.js` 是前端離線鏡像。
- 若修改 `.db`，必須重新執行：`tools/export_cache_from_db.py`

### 2.3 存檔
- 玩家進度存於瀏覽器本機 `localStorage`
- 匯出／匯入存檔由 `assets/save-system.js` 處理

## 3. 目前已實作核心系統
### 3.1 角色基礎
- 六大主屬性：STR / VIT / DEX / AGI / INT / LUK
- 有等級與經驗值
- 起始身分：**平民**
- 可透過培養方式就職副職業

### 3.2 技能系統
- 攻擊技能已拆分為：**武技**、**魔法**
- 另有：生活／支援、被動技能
- **初始下位技能改為透過對應行動機率習得**
- 技能有熟練度
- 熟練度滿後可由下位技能進化為上位技能
- 被動技能會影響能力值、判定或特殊效果
- 目前已實作的起手領悟技能：斬擊、火花、急救、鍛體、旅者直覺、魔物解剖

### 3.3 戰鬥系統
- 有基本戰鬥流程
- 已改回 **傳統 RPG 百分比判定**
- 戰鬥中可查看敵人狀態（HP、攻防、敏捷、類型、經驗等）

### 3.4 公會委託系統
- 有冒險者公會
- 可接受委託
- 已有魔物討伐委託
- 討伐委託可要求解剖指定部位後交付

### 3.5 魔物解剖系統
- 生物型魔物討伐後會留下屍體
- 可對屍體進行部位式解剖
- 解剖熟練度會影響戰利品品質與稀有素材取得
- 已有部位個別掉落與交付對應機制

### 3.6 生存系統
- 目前先實作：**飽食度**
- 飽食度會隨時間與冒險下降
- 過低會影響生存與戰鬥表現

### 3.7 副職業系統
目前已有生活副職業：
- 裁縫
- 鐵匠
- 廚師
- 農夫
- 獵人

## 5. 目前系統檔案分工
- `assets/core-system.js`：共享狀態、衍生數值、等級成長等核心工具
- `assets/skill-discovery-system.js`：初始技能的行動式機率習得
- `assets/skill-system.js`：技能熟練、技能進化、技能使用
- `assets/combat-system.js`：戰鬥流程、命中、敵人狀態、戰後屍體
- `assets/action-system.js`：世界互動與行動派發
- `assets/save-system.js`：本地存檔、匯出、匯入、存檔相容處理
- `assets/render-system.js`：UI 渲染
- `assets/dice-system.js`：傳統 RPG 百分比判定與命中計算
- `assets/guild-system.js`：公會委託接受、進度、交付
- `assets/anatomy-system.js`：屍體、部位解剖、掉落品質
- `assets/survival-system.js`：飽食度等生存項
- `assets/profession-system.js`：平民起始與副職業系統
- `assets/main.js`：初始化、事件綁定、整體進入點
- `assets/database.js`：資料快取讀取橋接
- `assets/db-cache.js`：由 `.db` 匯出的前端資料快取

## 9. 接續開發時必守規則
1. **離線單機優先**：不得要求額外後端或伺服器。
2. **數據類內容優先寫入 `.db`**：魔物、NPC、物品、技能、任務、地點、解剖部位、職業等都應先改資料庫。
3. **改完 `.db` 要同步更新 `assets/db-cache.js`**。
4. **各系統維持拆檔**：不要把邏輯重新混回單一大檔。
5. **若新增大型系統，優先新增獨立 JS 檔案**。
6. **若變更核心規則或資料表，必須更新本文件與 `docs/ai_handoff.json`。**
7. **若調整「行動習得技能」規則，需同步更新起手技能來源提示。**
8. **zip 交付時，保留這份文件，讓新對話 AI 可快速接手。**
